# UX v1.12.0 Integration Design

Date: 2026-07-16  
Branch: `codex/ux-1.12.0`  
Base: `origin/main` at released `v1.11.0` (`8da06d6`)  
Source material: local `feat/ux-1.11.0` (`5b0bc1a`)

## Goal

Ship one coherent `1.12.0` release that preserves every correctness, alerting,
storage, security, and deployment fix already released in `1.11.0`, while
adding the unfinished onboarding, CLI, MCP consistency, dashboard UX, and card
language work from the local UX branch.

The old UX commit is a reference, not a merge unit. Its implementations must
never overwrite newer `1.11.0` code merely because they touch the same file.

## Integration strategy

1. Keep `feat/ux-1.11.0` unchanged as a recovery point.
2. Build from `origin/main`, not from the stale `1.10.2` merge base.
3. Copy files that are genuinely new and independent.
4. Re-implement overlapping behavior against current `1.11.0` interfaces,
   retaining current helpers such as `withLocalDb`, `format-num`,
   `smartMoneyShareOfOI`, the alert fingerprint/cooldown logic, the shared
   symbol list, and the corrected batch timeout.
5. Delete duplicated fixes from the integration scope instead of carrying two
   versions of the same behavior.

## Included behavior

### Onboarding and documentation

- Add a complete, commented `.env.example`.
- Lead both READMEs with the zero-install MCP path, then separate the clone and
  tracker path for local time-series data.
- Use one canonical MCP install command everywhere:
  `claude mcp add smartmoney -- npx -y binance-smart-money-oi-monitor@latest`.
- Explain that tracker, dashboard, and MCP time-series tools must share the same
  absolute `SMART_MONEY_DB_PATH`.
- Keep English and Chinese environment-variable references equivalent.
- Add an automated documentation contract test for version parity, environment
  variable coverage, canonical commands, duplicate headings, and local anchors.

### CLI

- `change` and `trend` show a human table by default.
- `--json` retains machine-readable output; npm invocations document the
  required `--` separator, while `npm_config_json` remains supported.
- `--help`/`-h` exit successfully.
- Error results go to stderr and set a non-zero exit code.
- `doctor` prints a final READY/NOT READY verdict and fails only on blocking
  checks, not warnings.

### MCP and cards

- Add `zh|en` card rendering with precedence: explicit argument, then
  `SMART_MONEY_CARD_LANG`, then `zh`.
- Expose `lang` per call on `render_panel` and `render_push`.
- Preserve every numeric unit and scaling rule from `1.11.0`.
- Add ratio direction hints without changing numeric fields.
- Attach the disclaimer to editorial/interpretive outputs, including
  `get_change`, `get_profit_trend`, and `render_chart`; keep raw metric tools
  free of redundant disclaimer fields.

### Dashboard

- Preserve `1.11.0`'s canonical SM-share helper and safe formatting.
- Extend missing-DB detection to better-sqlite3's fresh-directory `TypeError`.
- Keep the friendly 503 onboarding page for both list and symbol HTML routes.
- Add a compact search field, zero-row guidance, legend/tooltips, load time,
  and an intuitive default sort without adding dependencies or a second UI
  framework.
- Keep all user-controlled values HTML-escaped and make the table usable on a
  375 px viewport through horizontal overflow rather than clipped content.

## Explicitly retained from v1.11.0

- Fresh-or-null forced fetch semantics and block handling.
- Alert fingerprinting, cooldown, from-zero triggers, short-side P&L, and send
  failure reporting.
- `withLocalDb`, guarded read-only DB access, price-column tolerance, and live
  connection reuse.
- Shared `symbol-list.ts` and the removal of bare axios discovery calls.
- Canonical `format-num` percentage/fraction separation.
- Correct batch timeout cleanup and timeout-only warning.
- Docker `oi-tick` service and the package tracker-bin documentation.

## Versioning and release metadata

`1.12.0` must appear consistently in `package.json`, `package-lock.json` (root
and package entry), MCP `SERVER_INFO`, the MCP version test, CHANGELOG, and the
release design documentation. Existing `1.11.0` history remains intact.

## Verification

- Red-green tests for each new behavior.
- Full `npm test`, `npm run typecheck`, and `npm run build`.
- `npm pack --dry-run` plus tarball-content inspection.
- Documentation contract test and Markdown link/heading scan.
- Dashboard route smoke test with a missing parent directory and a temporary DB.
- Final diff audit against `origin/main` to prove no `1.11.0` correctness files
  were deleted or reverted.

## Non-goals

- No new trading signals or buy/sell recommendations.
- No dependency-heavy screenshot renderer.
- No automatic locale detection.
- No changes to OI, funding, profit, ratio, or notional numeric semantics.
- No npm publish, GitHub push, tag, or release without separate authorization.
